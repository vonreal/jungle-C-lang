5. 페이지 폴트(Page Fault)의 개념을 설명하고, 페이지 폴트가 발생했을 때 운영체제가 수행하는 절차를 본인이 이해한 만큼 자세히 서술하시오.

나의 답)
페이지 폴트는 메인 메모리에서 참조하려는 페이지가 없을 때 발생한다.
PTE의 valid bit = 0일 경우 메인 메모리에 해당 페이지가 없는 것이고,
그럼 운영체제는 디스크에서 해당 페이지를 메인 메모리로 올리고 valid bit = 1로 올린다.
여기까지는 page를 아예 올리지 않았을 경우고,
victim page라는 것이 있다. 이미 한 번 올렸던 페이지이지만 LRU 등 사용하지 않는 페이지를 SWAP으로 디스크에 올리는데 선정되는 페이지가 있다.
맥과 같은 최신 운영체제에서는 memory compressed를 진행해서 메인 메모리의 공간에 victim page를 압축시켜놓는다.
(디스크보단 메모리에 압축된 걸 풀어서 가져오는게 더 빠르니까.)

🥕 보완점)
트랩/예외 발생 → CPU가 OS에 제어권을 넘기는 과정 언급이 없어
빈 프레임 확인 → 빈 프레임이 있으면 바로 할당, 없으면 victim 선정이라는 분기
victim page가 dirty bit가 1이면 디스크에 write-back, 0이면 그냥 교체하는 과정
페이지 테이블 갱신 후 중단된 명령어를 재실행한다는 점

그렇다면 굳이 mmap() 시스템 호출을 사용하여 메모리를 할당받는 이유는 무엇인지 설명하시오.

나의 답)
운영체제가 가상메모리에서 메모리공간을 할당해줘야하기 때문에?
시스템 호출은 유저 프로그램이 하드웨어를 사용하는 것을 운영체제가 대신 관리해주는 것인데,
물리 메모리 주소에 접근하지 않고 운영체제가 가상 메모리 주소로 할당해주기 때문에 mmap()을 사용한다고 추론해보았다.

👀 답)

대용량 할당: malloc은 힙에서 관리하는데, 큰 메모리는 mmap으로 OS에서 직접 페이지 단위로 받는 게 효율적 (실제로 glibc의 malloc도 큰 할당엔 내부적으로 mmap 사용)
프로세스 간 공유 메모리: MAP_SHARED로 여러 프로세스가 같은 물리 메모리를 공유
해제 즉시 OS 반환: free()는 힙에 반환하지만 OS에 바로 돌려주지 않을 수 있고, munmap()은 즉시 반환

좋아, malloc과 mmap을 나란히 비교해볼게.

**malloc은 라이브러리 함수, mmap은 시스템 콜이야.** 이게 가장 근본적인 차이인데, 이걸 중심으로 풀어보면:

**malloc의 동작 방식**

malloc은 C 라이브러리(glibc)가 관리하는 **힙 영역**에서 메모리를 잘라서 줘. 내부적으로는 처음에 sbrk()나 mmap()으로 OS에서 큰 덩어리를 받아온 다음, 그 안에서 요청 크기에 맞게 쪼개서 나눠주는 거야. 일종의 중간 관리자 역할이지. free()로 반환해도 라이브러리가 나중에 재사용하려고 들고 있을 수 있어서, OS에 바로 돌아가지 않는 경우가 많아.

**mmap의 동작 방식**

mmap은 OS 커널에 직접 요청해서 **페이지 단위로** 가상 주소 공간에 새로운 영역을 매핑해. 힙을 거치지 않고 OS가 직접 관리하는 거야.

**그래서 mmap을 직접 쓰는 이유는:**

첫째, **파일 매핑**. 이게 mmap의 본래 목적이야. 파일을 가상 메모리에 매핑하면 read()/write() 시스템 콜 없이 포인터 연산만으로 파일 내용에 접근할 수 있어. malloc으로는 이게 불가능해.

둘째, **프로세스 간 공유 메모리**. MAP_SHARED 플래그로 여러 프로세스가 같은 물리 페이지를 공유할 수 있어. malloc은 해당 프로세스의 힙 안에서만 동작하니까 이런 게 안 돼.

셋째, **즉시 OS 반환**. munmap() 하면 해당 페이지가 바로 OS에 돌아가. free()는 힙 관리자가 잡고 있을 수 있어서 메모리 사용량이 바로 안 줄어들 수 있고.

넷째, **큰 할당의 효율성**. 재밌는 건, glibc의 malloc도 보통 128KB 이상 요청이 들어오면 내부적으로 mmap을 호출해. 힙에서 큰 덩어리를 관리하는 것보다 OS에서 직접 페이지를 받는 게 단편화 면에서 나으니까.

**정리하면:**

malloc은 "작은 메모리를 빠르게 할당/해제"하는 범용 도구이고, mmap은 "파일 매핑, 공유 메모리, 대용량 할당처럼 OS 수준의 제어가 필요할 때" 쓰는 도구야. 용도 자체가 달라서, malloc으로 할 수 없는 일들이 있기 때문에 mmap이 필요한 거지.
